## Overview

Mastra's modular voice architecture allows you to use different providers for text-to-speech (TTS), speech-to-text (STT), and real-time voice interactions. You can select providers based on factors like voice quality, language support, latency, and special features.

## Choosing a Provider

### Selection Criteria

When selecting a voice provider, consider:

1. **Capability Requirements**
   - Do you need TTS, STT, or both?
   - Is real-time interaction necessary?

2. **Quality Factors**
   - How important is voice naturalness?
   - Do you need emotion or emphasis control?

3. **Technical Requirements**
   - What languages do you need to support?
   - What are your latency requirements?
   - Do you need specialized features like custom vocabulary?

4. **Business Considerations**
   - What is your budget?
   - What are the usage volumes?
   - Are there regulatory or compliance requirements?

### Provider Strengths

Here's a quick guide to help you select:

- **OpenAI**: Best all-around option with good quality and real-time capabilities
- **ElevenLabs**: Best for premium voice quality and emotional expression
- **Deepgram**: Best for real-time transcription and accuracy
- **Google**: Best for broad language support and integration
- **PlayAI**: Best for voice cloning and customization
- **Murf**: Best for studio-quality voice production
- **Sarvam**: Best for Indian languages

## Using Multiple Providers

One of Mastra's key strengths is the ability to combine different providers using `CompositeVoice`:

```typescript
import { CompositeVoice } from "@mastra/core/voice";
import { OpenAIVoice } from "@mastra/voice-openai";
import { ElevenLabsVoice } from "@mastra/voice-elevenlabs";

// Initialize individual providers
const openAIVoice = new OpenAIVoice({
  listeningModel: {
    name: 'whisper-1',
    apiKey: process.env.OPENAI_API_KEY,
  },
});

const elevenLabsVoice = new ElevenLabsVoice({
  speechModel: {
    name: 'eleven_multilingual_v2',
    apiKey: process.env.ELEVENLABS_API_KEY,
  },
  speaker: '9BWtsMINqrJLrRacOk9x', // Aria voice
});

// Combine the providers
const voice = new CompositeVoice({
  listeningProvider: openAIVoice,    // Use OpenAI for STT
  speakingProvider: elevenLabsVoice, // Use ElevenLabs for TTS
});
```

This approach lets you select the best provider for each function.

## Best Practices

### API Key Management

Always store API keys securely:

```typescript
// Use environment variables
const voice = new OpenAIVoice({
  speechModel: {
    apiKey: process.env.OPENAI_API_KEY,
  },
});

// In production environments, consider:
// - Using a secrets manager
// - Rotating keys regularly
// - Setting up usage alerts
```

### Error Handling

Implement robust error handling:

```typescript
try {
  const audioStream = await voice.speak("Hello world");
  // Handle success
} catch (error) {
  if (error.code === 'AUTH_ERROR') {
    // Handle authentication issues
  } else if (error.code === 'RATE_LIMIT') {
    // Handle rate limiting
  } else {
    // Handle other errors
    console.error("Voice provider error:", error);
  }
}
```

### Fallback Providers

For critical applications, implement fallback mechanisms:

```typescript
async function speakWithFallback(text) {
  try {
    // Try primary provider
    return await primaryVoice.speak(text);
  } catch (error) {
    console.warn("Primary voice provider failed, using fallback");
    // Fallback to secondary provider
    return await fallbackVoice.speak(text);
  }
}
```
